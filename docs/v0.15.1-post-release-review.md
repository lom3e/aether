# Post-Release Review - v0.15.1 Runtime Consistency Patch

## 1. Problema Individuato
Durante un'analisi post-release della milestone v0.15.0 è emerso che nel codice di Aether venivano utilizzati sia timestamp timezone-aware (con fuso orario UTC esplicito) sia timestamp naive tramite `datetime.utcnow()`.
Nello specifico, l'utilizzo di `datetime.utcnow()` è stato deprecato nelle versioni recenti di Python in quanto restituisce un oggetto naive, che rischia di creare disallineamenti o eccezioni di comparazione in fase di logging o persistenza (come tra le interfacce di Memoria e il nuovo livello di Observability). Inoltre, la generazione di un `ResourceWarning` durante l'esecuzione dei test segnalava che non venivano pulite correttamente alcune risorse HTTP nei test mock.

## 2. Soluzione Adottata
Tutti gli utilizzi di `datetime.utcnow()` nel codice sono stati convertiti alla direttiva raccomandata `datetime.now(timezone.utc)`. 
È stato aggiunto anche un `exc.close()` nel blocco try/finally del gestore errori HTTP in `OllamaProvider` per chiudere l'oggetto in modo esplicito e risolvere l'unraisable `ResourceWarning` durante le fasi di garbage collection dei test.

## 4. Test Eseguiti
La suite completa dei test è stata eseguita con la configurazione stringente sui warning (`pytest -W error tests/ -m "not integration" -q`).
È stato integrato un test specifico `tests/test_datetime_consistency.py` che assicura che il `MemoryDocument` produca nativamente timestamp timezone-aware (UTC) e che il sistema di storage persistente SQLite `SemanticMemory` mantenga tale consapevolezza lungo i cicli di serializzazione e deserializzazione.
Tutti i 144 test passano ora con zero regressioni e zero warning.

## 5. Conferma Compatibilità Retroattiva
L'architettura generale e la milestone v0.15.0 rimangono totalmente inalterate. L'adozione nativa di timezone-aware non introduce breaking changes nelle API pubbliche, né altera la struttura dei payload JSON salvati, dal momento che il formato `.isoformat()` gestisce in automatico l'aggiunta del suffisso `+00:00` (UTC) rendendo la stringa testuale persistita standard e pienamente retrocompatibile. Nessun utente esterno risentirà del cambiamento, se non beneficiando della completa eliminazione di _DeprecationWarning_.
